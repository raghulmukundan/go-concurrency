Concurrency Vs Parallelism

Concurrency

Concurrency is the ability of a system to handle multiple tasks at the same time. It allows different parts of a program to be executed out of order or in partial order, without affecting the final outcome. Concurrency is achieved by interleaving the execution of tasks, often using techniques like multitasking, multithreading, or asynchronous programming.

Key Characteristics of Concurrency:

  • Task Interleaving: Tasks are broken into smaller chunks, and the system switches between them.
  • Non-deterministic Order: The order of execution is not guaranteed.
  • Resource Sharing: Tasks share resources (e.g., CPU, memory) and may need synchronization.
  • Focus on Responsiveness: Concurrency is often used to improve the responsiveness of a system (e.g., handling multiple user requests in a web server).
